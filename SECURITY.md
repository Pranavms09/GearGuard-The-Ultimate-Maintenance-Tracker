# 🔐 Security Policy

## 📌 Supported Versions

We actively maintain and provide security updates for the latest version of GearGuard.

| Version | Supported |
| ------- | --------- |
| Latest  | Yes       |
| Older   | No        |

---

## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

**Do NOT create a public GitHub issue for security-related reports.**

Instead, you can:

- Use GitHub's **Security Advisory** feature (if available)
- Or contact the project maintainers directly via GitHub

### Please include the following details:

- Clear description of the vulnerability  
- Steps to reproduce the issue  
- Potential impact  
- Suggested fix (if applicable)  

---

## ⏱️ Response Timeline

| Action          | Expected Time      |
| --------------- | ------------------ |
| Acknowledgement | Within 48–72 hours |
| Initial Review  | Within 5 days      |
| Resolution      | Based on severity  |

---

## 🔒 Security Best Practices

To ensure the security of GearGuard:

- Never commit `.env` files or sensitive credentials  
- Use `.env.example` as a reference  
- Validate and sanitize all user inputs (client & server)  
- Keep dependencies updated (`npm audit`)  
- Follow secure authentication practices  
- Restrict access to sensitive routes and APIs  

---

## 📢 Disclosure Policy

We follow responsible disclosure:

- Issues are fixed before public disclosure  
- Fixes are verified before release  
- Details may be shared after resolution  

---

## 🧩 Scope

This policy applies to:

- Frontend (client)  
- Backend (server)  
- API endpoints  
- Database and environment configuration  

---

## 🙌 Acknowledgement

We appreciate responsible reporting of vulnerabilities.

Thank you for helping keep GearGuard secure.
